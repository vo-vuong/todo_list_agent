## Implementation Notes

- Streamlit callbacks use a small `run_async()` wrapper around the existing async MCP/planner helpers, matching the CLI's `anyio.run()` pattern.
- Task actions do not read or write `data/tasks.json` directly from the UI; all task list/add/complete/reset actions go through MCP client helpers.
- Browser plugin verification was attempted, but the in-app browser returned `Browser is not available: iab`; verification continued with Streamlit HTTP smoke and Streamlit `AppTest`.
- The Phase 2 non-blocking P2 finding about post-LLM task membership/priority validation remains known. Phase 3 did not claim stronger planner validation than currently implemented.

## Deviations From Plan

- Streamlit browser verification used Streamlit `AppTest` and HTTP smoke because the in-app Browser surface was unavailable in this session.
- The MCP client connection wrapper was narrowed while adding UI helpers. This also addresses the Phase 2 P3 review finding about tool-call errors being mislabeled as connection failures.
