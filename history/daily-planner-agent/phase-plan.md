# Daily Planner Agent - Phase Plan

**Feature slug:** daily-planner-agent
**Date:** 2026-06-10
**Mode:** standard_feature
**Source of truth:** `history/daily-planner-agent/CONTEXT.md`

## Feature Summary

Build a small Python Streamlit demo where a daily planner agent fetches and mutates tasks through a separate FastMCP HTTP/SSE server backed by local JSON. The agent uses PydanticAI with OpenAI to prioritize today's incomplete tasks and write a concise dated markdown schedule. The demo must remain repeatable through sample data and reset behavior, with `uv` setup and clear missing-key failure in the UI.

## Why Phase-Shaped Work

Direct or small work is insufficient because the feature spans a new Python project scaffold, an MCP server process, JSON persistence, agent orchestration, LLM-dependent scheduling, CLI behavior, report export, and Streamlit UI workflows. Phases keep each milestone demonstrable without turning the plan into technical buckets.

## Order Check

- First phase is obvious because the separate MCP task boundary is the shared dependency for CLI, agent, and Streamlit workflows.
- Later phases build on earlier observable outcomes: Phase 2 consumes the MCP task loop, and Phase 3 exposes the proven task and planner flows in the UI.
- No phase is just a technical bucket such as "backend", "frontend", or "tests"; each phase has a runnable user-visible proof.
